# Archive Image Viewer & Extractor — 完全仕様書

## 1. プロジェクト概要

### 1.1 目的
ZIP/RAR/7z等で圧縮された画像アーカイブの中身をサムネイルグリッドで一覧表示し、範囲選択して別ZIPに再圧縮（抽出）したり、アーカイブ自体を評価して指定フォルダへ移動/コピーする仕分けツール。動画ファイルの再生・仕分け機能も備える。

### 1.2 動作環境
- **OS**: Windows専用（`ctypes.windll`, `os.startfile`, `windnd` 等Windows APIに依存）
- **Python**: 3.10+（`run_archive_viewer.bat`でPython 3.10のpythonw.exeを直接指定）
- **GUI**: tkinter（標準ライブラリ）

### 1.3 ファイル構成

```
archive_viewer/
├── archive_viewer.py      # 全ロジック・UI（単一ファイル、約4360行）
├── icon.ico               # アプリアイコン（.ico形式）
├── icon.jpg               # アプリアイコン（JPG形式、未使用）
├── requirements.txt       # 依存パッケージ一覧
├── run_archive_viewer.bat # 起動用バッチファイル
├── .gitignore
└── docs/                  # ドキュメント
```

### 1.4 依存パッケージ

| パッケージ | 必須/任意 | 用途 |
|-----------|----------|------|
| `Pillow>=9.0` | **必須** | 画像の読み込み・リサイズ・サムネイル生成 |
| `rarfile>=4.0` | 任意 | RAR形式の読み込み |
| `windnd>=1.0` | 任意 | ドラッグ&ドロップ対応（Windows） |
| `python-vlc>=3.0` | 任意 | 動画再生（VLC media playerのインストールが別途必要） |

### 1.5 外部ツール依存
- **7-Zip**: `C:/Program Files/7-Zip/7z.exe` — 7z/RAR(rarfileフォールバック)形式の展開に使用
- **VLC media player**: 動画モードで必要（python-vlcがDLLを参照）

## 2. アプリケーション構造

### 2.1 起動フロー

```
__main__ → check_dependencies() → ArchiveViewer() → mainloop()
```

1. `check_dependencies()`: Pillowが無ければエラーダイアログで終了。rarfile未インストールは警告のみ
2. `ArchiveViewer.__init__()`:
   - tk.Tkを継承してウィンドウ生成（1400x850、最小800x600）
   - AppUserModelID設定（タスクバーアイコン用）
   - 設定ファイル読み込み (`_load_config`)
   - UI構築 (`_build_ui`)
   - キーバインド設定 (`_bind_keys`)
   - ドラッグ&ドロップ設定 (`_setup_dnd`)
   - 前回モードの復元

### 2.2 起動バッチ
`run_archive_viewer.bat`: `pythonw.exe`で起動（コンソールウィンドウ非表示）

## 3. 3つの動作モード

アプリには3つのモードがあり、ヘッダーのボタンで切り替える。モードは終了時に保存され、次回起動時に復元される。

### 3.1 閲覧モード（browse）
- アーカイブ内の画像をサムネイルグリッドで一覧表示
- 画像の範囲選択 → ZIP抽出
- アーカイブ単位でフォルダへ移動/コピー（評価アクション）
- サイドバー: 左のみ

### 3.2 抽出モード（extract）
- 閲覧モードと同じグリッド表示
- 複数の範囲を「抽出リスト」に追加し、一括でZIP保存
- 各抽出エントリに著者名・作品名・話数のメタデータ入力可能
- サイドバー: 左（設定用）+ 右（抽出リスト用）

### 3.3 動画モード（video）
- VLCを使った動画再生
- シークバー、スキップボタン、音量調整
- 動画ファイル単位でフォルダへ移動/コピー
- サイドバー: 左のみ

## 9. 評価アクションシステム

### 9.1 概要
各モードにアクションボタンを配置し、現在のファイル（アーカイブ/動画）を指定フォルダへ移動またはコピーする仕分け機能。

### 9.2 アクションのデータ構造

```python
{
    "label": "撃墜",        # ボタンに表示するラベル
    "color": "#ef4444",     # ボタンの背景色
    "folder": "D:/path",    # 移動先フォルダ
    "copy": False           # True=コピー, False=移動
}
```

### 9.3 デフォルトアクション（5つ）

| ラベル | 色 | 用途（想定） |
|--------|-----|------|
| 撃墜 | 赤 (#ef4444) | 低評価 |
| 削除 | 暗赤 (#7f1d1d) | 削除対象 |
| ステイ | 青 (#3b82f6) | 現状維持 |
| 昇格 | 緑 (#22c55e) | 高評価 |
| 2段階昇格 | 黄 (#f59e0b) | 最高評価 |

### 9.4 移動/コピーの処理フロー

1. 移動先フォルダの存在確認
2. 同名ファイルが存在する場合 → `_resolve_conflict` ダイアログ
   - 上書き / 両方残す（リネーム）/ スキップ の3択
3. `_move_or_copy(src, dst, is_copy)` でファイル操作
   - `shutil.copy2`（コピー）または `shutil.move`（移動）
   - 移動後に `os.utime(dst)` で更新日時を現在に設定
4. 移動の場合 → `_after_file_action()` で次のファイルを自動表示

### 9.5 次ファイル自動表示の挙動（`_after_file_action`）
- 名前順: 元ファイルの次の名前のファイルを表示
- 日付順/ランダム: 同じインデックス位置のファイルを表示
- ビューアー表示中: ビューアーモードのまま次アーカイブの先頭を表示

### 9.6 動画モードの移動/コピー
- 大容量ファイル対応のため、プログレスバー付きバックグラウンド処理
- `_move_or_copy_with_progress`: 4MBチャンクでの読み書き
- 同一ドライブなら `Path.rename()`（高速）、別ドライブならチャンクコピー+元削除

## 10. ファイルナビゲーション

### 10.1 概要
同一フォルダ内のファイル（アーカイブまたは動画）間を前後に移動する機能。

### 10.2 操作方法
- キーボード: `A` / `←` (前) , `D` / `→` (次)
- ヘッダーの ◀ / ▶ ボタン

### 10.3 ファイル一覧のソート順（3種）

| ソート | 動作 |
|--------|------|
| 名前順 | Windows Explorer準拠（`StrCmpLogicalW`）。数字を自然順でソート |
| 更新日順 | ファイルの更新日時降順 |
| ランダム | 初回シャッフル後、ファイル追加/削除があっても順序を維持 |

- アーカイブ間を移動しても、ビューアーは閉じずに先頭ページを表示

## 11. サムネイル生成

### 11.1 2段階キャッシュ

1. **`thumb_data`**: 各画像を480×480以下にリサイズしたJPEGバイト列（品質85）
   - アーカイブ読み込み時に並列生成（`ThreadPoolExecutor`, CPU数×最大8ワーカー）
   - RGBA画像は背景色 (30,30,46) と合成
2. **`thumbnails`**: `thumb_data`から現在の`thumb_size`に合わせて生成した`ImageTk.PhotoImage`
   - ズーム操作や向き変更のたびに再生成

### 11.2 カード向きによるサイズ

| 向き | サムネイル最大サイズ |
|------|-------------------|
| 縦（portrait） | 幅 = size×2/3, 高さ = size |
| 横（landscape） | 幅 = size, 高さ = size×2/3 |

### 11.3 グリッドレイアウト
- カード幅 = サムネイル最大幅 + 32px
- 列数 = canvas幅 ÷ カード幅（最低1列）
- ウィンドウリサイズ時に自動再計算

## 12. アーカイブ読み込み

### 12.1 対応形式と優先順位

| 拡張子 | 読み込み方法 |
|--------|------------|
| `.zip` | `zipfile.ZipFile` |
| `.rar`, `.cbr` | `rarfile.RarFile` → 失敗時 `SevenZipArchive` |
| `.7z`, `.cbz` | `SevenZipArchive`（7z.exe CLIフォールバック） |

### 12.2 ファイルフィルタリング
- `__MACOSX`で始まるエントリを除外（ZIP）
- 画像拡張子（`SUPPORTED_EXTENSIONS`）のみ抽出
- Windows Explorer準拠でソート

### 12.3 読み込みフロー

```
_start_loading()
  → プログレスバー表示
  → スレッド起動: _load_archive()
    → _open_archive_file()  ← アーカイブを開く
    → ThreadPoolExecutor で並列サムネイル生成
      → _make_thumb(): 1枚ずつ読み込み→縮小→JPEG変換
      → 5枚ごとにプログレスバー更新
    → af.close()
    → after(0): _on_archive_loaded(thumb_data)
      → _build_thumbnails()  ← PhotoImage生成
      → _rebuild_grid()      ← カードをグリッド配置
```

## 13. 画像抽出

### 13.1 単発抽出（閲覧モード: `extract_range`）
1. 範囲選択（クリック + Shift+クリック）
2. `NameInputDialog`で命名
3. 保存先をファイルダイアログで指定
4. 選択範囲の画像をZIP_DEFLATEDで保存
5. ファイル名はフラット化（ディレクトリ階層を除去して`Path(name).name`のみ）

### 13.2 一括抽出（抽出モード: `_execute_extract`）
1. 複数の範囲を「抽出リスト」に追加
2. 各エントリに著者名・作品名・話数を入力（任意）
3. 「抽出実行」で全エントリを一括ZIP化
4. 出力先は事前に設定した`extract_output_folder`
5. ファイル名重複時は `_stem_2.zip`, `_stem_3.zip` ... と連番付加

### 13.3 抽出エントリのデータ構造

```python
{
    "start": 0,           # 開始インデックス
    "end": 9,             # 終了インデックス
    "author": "著者名",    # 任意
    "title": "作品名",     # 任意
    "episode": "第1話",    # 任意
    "extracted": False     # 抽出済みフラグ
}
```

## 14. 動画プレーヤー

### 14.1 VLC統合
- `python-vlc`ライブラリ経由
- VLCインスタンスオプション: `--no-xlib`, `--quiet`, `--no-video-title-show`
- 描画先: `video_canvas`の`winfo_id()`をHWNDとして`set_hwnd()`
- マウス入力・キー入力はVLCでは無効化（tkinterで処理）

### 14.2 遅延インポート
VLCモジュールはGIL競合回避のため`_init_vlc()`で必要時のみインポート。動画モード切替時に初めてロードされる。

### 14.3 再生制御

| 操作 | 実装 |
|------|------|
| 再生/一時停止 | `_video_play_pause()` — 画面クリックまたは▶ボタン |
| スキップ | `_video_skip(seconds)` — ±10s, ±30s, ±3m, ±10m |
| シーク | カスタムCanvas描画シークバー（ドラッグ対応） |
| 音量 | カスタムCanvas描画音量バー（ドラッグ対応、0〜100%） |
| ホイール | 動画の1%分スキップ |

### 14.4 再生位置更新ループ
`_video_update_loop()`: 300msごとに`after()`で再帰呼び出し。シークバー位置・時間表示・再生ボタン状態を更新。

### 14.5 再生終了(Ended)状態の処理
VLCのEnded/Stopped状態では通常のシーク/スキップが効かないため、`_video_restart_from(position)`でメディアを再セットして指定位置から再生する。

### 14.6 動画の閉じ方
- `_stop_video()`: `set_media(None)`で再生停止（`stop()`はPaused状態でハングするため未使用）
- `_release_video()`: プレーヤーを完全解放（モード切替時）
- `_close_video()`: UI初期化して動画ファイル選択待ちに戻る

## 20. ファイル競合ダイアログ（`_resolve_conflict`）

同名ファイルが移動先に存在する場合のモーダルダイアログ。

表示内容:
- ファイル名
- 移動元サイズ / 移動先サイズ
- サイズ比較（「移動元が大きい」「移動先が大きい」「同じサイズ」）

選択肢:
- **上書き（移動元で置換）**: そのまま上書き
- **両方残す（リネーム）**: `_make_unique_path`で`ファイル名 (1).ext`形式
- **スキップ（移動しない）**: 何もしない

## 23. 再現に必要な実装ポイント

### 23.1 メモリ管理
- 元画像データは即解放（`del data`）
- サムネイルは480px JPEGバイト列としてメモリ上にキャッシュ
- PhotoImageはzoomのたびに再生成

### 23.2 スレッド設計
- アーカイブ読み込み: `threading.Thread`（daemonスレッド）
- サムネイル生成: `ThreadPoolExecutor`（CPU数 or 最大8ワーカー）
- UIスレッド同期: `self.after(0, callback)` でメインスレッドに戻す
- プログレスバー更新: 5枚ごとに`after(0)`で更新

### 23.3 VLCの注意点
- `player.stop()`はPaused状態でハングする → `set_media(None)`を使用
- Ended状態ではシークが効かない → メディア再セット+位置指定で対応
- 遅延インポートでGIL競合を回避
- 動画のDnDはGIL競合のため無効化

### 23.4 Windows固有の実装
- `StrCmpLogicalW`: ファイル名の自然順ソート
- `SetCurrentProcessExplicitAppUserModelID`: タスクバーアイコン
- `os.startfile`: エクスプローラーでフォルダを開く
- `winfo_id()` + `set_hwnd()`: VLC描画先の指定
- `windnd`: ドラッグ&ドロップ
- CP932フォールバック: 日本語ファイル名のエンコーディング

### 23.5 カスタムシークバー / 音量バー
tkinterのScaleウィジェットではなく、Canvasに直接描画。マウスイベント（Click/Drag/Release）で操作。ノブは円形、トラックは矩形。
