# 画像閲覧モード（Image Browse Mode）— 機能仕様書

## 1. 概要

### 1.1 目的
フォルダ内の画像ファイル（非アーカイブ）を、既存の閲覧モードと同様のサムネイルグリッドで一覧表示し、**画像1枚単位**で指定フォルダへ移動/コピーできる仕分けモード。

### 1.2 既存モードとの違い

| 項目 | 閲覧モード（browse） | 画像閲覧モード（image） |
|------|----------------------|------------------------|
| 対象ファイル | アーカイブ内の画像 | フォルダ内の画像ファイル |
| ファイル読み込み | アーカイブを展開 | ファイルシステムから直接読み込み |
| サムネイル一覧 | アーカイブ内画像の一覧 | フォルダ内画像の一覧 |
| ナビゲーション | アーカイブ間を移動 | なし（フォルダ内画像を一括表示） |
| アクション対象 | アーカイブファイル全体 | **画像1枚** |
| ランダム移動 | アーカイブのランダム表示順 | 画像のランダム表示順 |
| ビューアー | アーカイブ内画像をページ送り | フォルダ内画像をページ送り |

### 1.3 モード識別子
- 内部名: `"image"`
- 表示名: `画像`

## 2. UI構成

### 2.1 ヘッダー

```
[閲覧] [抽出] [動画] [画像]  |  [名前順] [更新日順] [ランダム]  |  [縦] [横]  |  [📁 開く]  |  (右寄せ) フォルダパス表示
```

- モード切替ボタンに「画像」を追加（4つ目のモードボタン）
- ソート切替：フォルダ内の画像ファイルの表示順を変更
  - 名前順: NaturalStringComparer（Windows Explorer準拠）
  - 更新日順: ファイル更新日時降順
  - ランダム: シャッフル
- カード向き切替: 既存と同様（縦/横）
- 「開く」ボタン: **フォルダ選択ダイアログ**を表示（ファイル選択ではなく）
- ファイルナビゲーション（◀ ▶）: **非表示**（アーカイブ間移動は不要）
- 右寄せ部分: 現在表示中のフォルダパスを表示

### 2.2 左サイドバー

```
作業フォルダ表示（現在開いているフォルダパス）
─────────────
プリセット名表示
[保存] [管理]
─────────────
移動アクション  [+追加]
  [お気に入り] [複製] [📁] [✎]
    → 移動先パス表示
  [削除] [複製] [📁] [✎]
    → 移動先パス表示
  ...
─────────────
ソースフォルダ  [+追加]
  [▲▼] [📂 フォルダ名] [🎲] [×]
  ...
─────────────
ゴミ箱フォルダ
  [パス表示] [...] [📂]
```

- 閲覧モードのサイドバーと同構造
- ソースフォルダ: 画像ファイルを含むフォルダのリスト
- アクションボタン: クリック時に**現在選択中の画像1枚**を移動/コピー

### 2.3 メインエリア

- 閲覧モードと同一のサムネイルグリッド表示
- `ImageCard` コントロールで各画像を表示
- クリック: 画像の選択（単一選択のみ。範囲選択は不要）
- ダブルクリック: 画像ビューアーを開く

### 2.4 右サイドバー

- **非表示**（抽出機能なし）

### 2.5 画像ビューアー

- 既存のビューアーオーバーレイをそのまま使用
- フォルダ内の画像をページ送りで閲覧
- キーボード（A/D, ←/→）、マウスホイールで移動
- スライダーバーで任意の画像にジャンプ

### 2.6 ステータスバー

- 左: ステータスメッセージ
- 右: 画像枚数表示（例: 「画像: 42枚」）

## 3. データ構造

### 3.1 インスタンス変数（新規）

| 変数名 | 型 | 説明 |
|--------|-----|------|
| `_imageFolderPath` | `string?` | 現在開いている画像フォルダのパス |
| `_imageFolderFiles` | `List<string>` | フォルダ内の画像ファイルパスリスト（ソート済み） |
| `_selectedImageIndex` | `int?` | 現在選択中の画像インデックス |

### 3.2 プリセット（設定）

既存の3モード（browse / extract / video）と同様に、独立したプリセット辞書を持つ。

```csharp
// AppConfig に追加
public Dictionary<string, PresetData> ImagePresets { get; set; }

// AppState に追加
public string ImageCurrentPreset { get; set; }
```

### 3.3 設定ファイル（JSON）

```json
{
  "presets": { ... },
  "extract_presets": { ... },
  "video_presets": { ... },
  "image_presets": {
    "プリセット名": {
      "actions": [
        {
          "label": "お気に入り",
          "color": "#22c55e",
          "folder": "D:/favorites",
          "copy": true
        }
      ],
      "source_folders": ["D:/photos", "D:/downloads/images"],
      "trash_folder": "D:/trash"
    }
  },
  "image_current_preset": "プリセット名",
  "last_mode": "image"
}
```

## 4. 機能詳細

### 4.1 フォルダを開く

1. 「開く」ボタンまたはソースフォルダボタンをクリック
2. フォルダ選択ダイアログを表示（`System.Windows.Forms.FolderBrowserDialog` または同等）
3. フォルダ内の画像ファイルを拡張子フィルタで取得
   - 対象拡張子: `.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.webp`, `.tiff`（`Theme.ImageExtensions`）
   - サブフォルダは**対象外**（フォルダ直下のみ）
4. 現在のソート順でソート
5. サムネイル生成 → グリッド表示

### 4.2 画像ファイルの読み込み

既存の `LoadArchive()` とは別に、`LoadImageFolder()` メソッドを新設。

```
LoadImageFolder(folderPath):
  1. フォルダ内の画像ファイルをリストアップ
  2. ソート順を適用
  3. 各画像に対してサムネイルを並列生成
     - ファイルシステムから直接読み込み（アーカイブ展開不要）
     - ThumbnailService.GenerateThumbnailBytes() を使用
  4. グリッドに表示（RebuildGrid）
  5. ステータスバーに画像枚数を表示
```

**進捗表示**: 既存の `ProgressOverlay` を使用。画像数が多い場合はプログレスバーを表示。

### 4.3 サムネイルグリッド

- 既存の `RebuildGrid()` をそのまま流用
- `_imageNames` と `_thumbData` / `_thumbnails` に画像フォルダのデータを格納
- `ImageCard` のクリック動作:
  - **シングルクリック**: その画像を選択状態にする（単一選択）
  - **ダブルクリック**: 画像ビューアーを開く
- 範囲選択（Shift+クリック）: **無効**（画像1枚単位の操作のため不要）

### 4.4 画像の選択

- 1枚のみ選択可能（排他選択）
- 選択中の画像は紫色ボーダーでハイライト（`SELECT_COLOR`）
- 選択状態はアクション実行後にクリアされる

### 4.5 アクション実行（移動/コピー）

アクションボタンクリック時の処理フロー:

```
ExecuteImageAction(action):
  1. 選択中の画像がなければ何もしない
  2. src = _imageFolderFiles[_selectedImageIndex]
  3. dst = action.Folder / ファイル名
  4. フォルダ作成（必要なら）
  5. 同名ファイル存在チェック → ConflictDialog（上書き/リネーム/スキップ）
  6. コピーまたは移動を実行
  7. File.SetLastWriteTime(dst, DateTime.Now)
  8. 移動の場合:
     a. _imageFolderFiles からエントリを除去
     b. サムネイルデータから該当エントリを除去
     c. グリッドを再構築（RebuildGrid）
     d. 次の画像を自動選択（同じインデックス位置、範囲外なら末尾）
  9. コピーの場合:
     a. リストはそのまま維持
     b. ステータスバーにコピー完了メッセージ
```

### 4.6 移動後のグリッド更新

- 画像を移動した場合、その画像をグリッドから即座に除去
- **フォルダの再スキャンはしない**（パフォーマンスのため、内部リストから除去するのみ）
- 残り画像が0枚になったらEmptyMessageを表示

### 4.7 ビューアーモード中のアクション

- ビューアー表示中にアクションボタンをクリックした場合:
  - 現在ビューアーで表示中の画像に対してアクションを実行
  - 移動の場合、次の画像を自動表示（ビューアーは閉じない）
  - 最後の画像を移動した場合、ビューアーを閉じる

### 4.8 ソート順

| ソート | 動作 |
|--------|------|
| 名前順 | NaturalStringComparer（Windows Explorer準拠） |
| 更新日順 | ファイル更新日時降順 |
| ランダム | シャッフル。ファイル移動で除去されても他の順序は維持 |

ソート切替時: 内部リストを再ソートし、サムネイルデータも連動して並び替え → グリッド再構築。

### 4.9 ドラッグ＆ドロップ

- フォルダをウィンドウにドロップ → 画像閲覧モードで開く
- 画像ファイルをドロップ → そのファイルの親フォルダを画像閲覧モードで開き、該当画像を選択状態にする

## 5. キーボード操作

| キー | 動作 |
|------|------|
| `A` / `←` | ビューアー中: 前の画像 |
| `D` / `→` | ビューアー中: 次の画像 |
| `Escape` | ビューアーを閉じる |
| `Delete` | 選択中の画像をゴミ箱フォルダへ移動 |
| マウスホイール | ビューアー中: 画像送り |
| `Ctrl` + マウスホイール | サムネイルサイズ変更 |

## 6. モード切替時の動作

### 6.1 画像モードに切り替え

- 動画再生中なら停止
- アーカイブのサムネイルグリッドを非表示
- 画像フォルダが読み込み済みなら画像グリッドを表示、未読み込みならEmptyMessage表示
- ナビゲーションボタン（◀ ▶）を非表示
- 抽出関連UI（範囲抽出ボタン、右サイドバー）を非表示
- サイドバーを画像モード用に再構築

### 6.2 画像モードから他モードへ切り替え

- 画像グリッドは内部データを保持（メモリ上にキャッシュ）
- 戻ってきた時に再表示可能

## 7. 実装方針

### 7.1 既存コードの流用

以下の既存機能はほぼそのまま流用可能:

| 機能 | 流用元 | 変更点 |
|------|--------|--------|
| サムネイルグリッド | `RebuildGrid()` | なし |
| `ImageCard` コントロール | `Controls/ImageCard.xaml` | なし |
| サムネイル生成 | `ThumbnailService` | なし（byte[]入力を流用） |
| 画像ビューアー | `ViewerOverlay` | 画像読み込みをファイルから直接に |
| プリセット管理 | `PresetManagerDialog` | なし |
| アクション設定 | `SettingsDialog` | なし |
| ConflictDialog | `Dialogs/ConflictDialog` | なし |
| ProgressDialog | `Dialogs/ProgressDialog` | なし |
| 移動/コピー処理 | `CopyFileWithProgress` / `MoveFileWithProgress` | なし |

### 7.2 新規実装が必要な箇所

| 機能 | 説明 |
|------|------|
| `LoadImageFolder()` | フォルダ内画像の読み込みとサムネイル生成 |
| `ExecuteImageAction()` | 画像1枚の移動/コピー + グリッド更新 |
| `AfterImageAction()` | 移動後のリスト更新とグリッド再構築 |
| `SwitchMode("image")` | モード切替ロジックへの追加 |
| `RebuildSidebar()` の `case "image":` | サイドバー構築 |
| `AppConfig` / `AppState` への追加 | プリセット・状態の永続化 |
| ヘッダーへの「画像」ボタン追加 | XAML変更 |
| ビューアーの画像読み込み | アーカイブからではなくファイルから直接読み込み |

### 7.3 変更が必要な既存コード

| ファイル | 変更内容 |
|---------|----------|
| `MainWindow.xaml` | 「画像」モードボタン追加、ナビゲーション表示制御 |
| `MainWindow.xaml.cs` | SwitchMode拡張、LoadImageFolder、ExecuteImageAction、AfterImageAction、ビューアー読み込み分岐 |
| `Models/AppConfig.cs` | ImagePresets, ImageCurrentPreset 追加 |
| `Services/ConfigService.cs` | 新プリセット辞書の読み書き |
| `Theme.cs` | 必要に応じて定数追加 |

## 8. 注意事項・制約

- **サブフォルダは対象外**: フォルダ直下の画像ファイルのみ。再帰的な探索はしない。
- **大量画像への対応**: フォルダ内に数百〜数千枚の画像がある場合、サムネイル生成に時間がかかる。既存のプログレスバーとキャンセル機能を流用する。
- **ファイル監視はしない**: 外部で画像が追加/削除されても自動更新しない。手動でフォルダを再読み込みする必要がある。
- **アーカイブファイルは無視**: フォルダ内に `.zip` 等があっても画像モードでは表示しない。画像拡張子のファイルのみ。
